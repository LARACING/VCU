version 0.20.1  prelim Date: 2016-07-06
---------------------------------------
- DM1 handler counts up the occurrence count only on the positive edge of the trigger respectively in the moment
  when an FMI is to be notified.
NOTE: When a fault occurs as a short glitch (shorter than the DM1 send-period) the occurrence count will be counted 
up internally, but as the fault has vanished no DM1 will be send out. Consequently, if the corresponding DM1 
messages shows up later on due to the same but longer lasting fault (longer than the DM1 send-period) the 
occurrence count will visibly have been counted up by two. This behaviour will be corrected in an upcoming release.


-------------------------------------
version 0.20  prelim Date: 2016-06-06
-------------------------------------
- The block MainDlg4App reads out EEPROM (Offset 0x0..0x2) and IO_DI_52..54 in order to extract application coding
  In case information retrieved from EEPROM does not match with the DI the new application number is written to EEPROM
  and the default application is called. The start the new application regularily a hard reset (KL30) is necessary.
NOTE: This release is only ment as a prliminary work around. In future only IO_DI_00..35 may be used for coding information.

version 0.16  Date: 2015-11-09  
------------------------------
- Blocks to simply access the EEPROM
- EEPROM-Importer capable to generate a library based on the given EEPROM layout-file.
- hex_read and hex_write for reading & writing *.ihex files handled by the TTC-Downloader
  Note: TTC-Downloder may only handle 32k of EEPROM in IO-Driver version 2.8.0. 
  Please use in EEPROM-Impoter only EEPROM 0x0000 to 0x7FFF. This will be fixed in next release of IO-Driver.
- Write_RS232 support for int8, int16, int32 (correct display of negative numbers) 
- DBC-Importer: debounce semantics changed, with the 2nd consecutive error all errors are displayed regardless
                whether these are identical or not. This sequence is interrupted by IO_E_OK and resumed by 
                two successive errors.
- DBC-Importer: a event message in the dbc-file is mapped to a regular (non-debounced) CAN Message block in order
                to get an instandly reported ErrorCode.
- Diag_Status: Resolved link status to block "compare to zero"
